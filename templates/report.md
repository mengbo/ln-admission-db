# 志愿填报方案

> 本模板用于 AI Agent 在收到考生信息后一次性生成完整志愿填报方案。
> 拿到 [question.md](./question.md) 填写完的内容后，按本模板的章节顺序输出。
>
> 方案与具体年份无关：
> - "今年" = 考生参加高考的年份（AI 从考生信息中获取）
> - "去年" = 今年 − 1
> - "前年" = 今年 − 2
> - 历史分数与计划数取自 `score_phyYYYY` / `score_hisYYYY` / `plan_YYYY`，YYYY 对应实际年份
>
> **下方括号内的年份（如 2026 / 2025 / 2024）均为示例，AI 须按考生实际高考年份替换为对应今年/去年/前年。**
>
> **关于名称**：本项目把"志愿填报报告 / 志愿填报指南 / 志愿方案 / 报告 / 方案 / 指南"都视为同义词，最终输出物的正式名称是 **"志愿填报方案"**。详见 [AGENTS.md 术语约定](../AGENTS.md#-术语约定)。

## 1. 定位分析

> 等效分换算的原理与计算步骤见 [AGENTS.md 数据库分析经验](../AGENTS.md#数据库分析经验)。

### 等效分换算表

| 年份 | 分数 | 位次 | 说明 |
|---|---|---|---|
| 今年（如 2026） | 考生原始分 | 考生位次（从今年 rank 表查） | 直接给出 |
| 去年（如 2025） | 去年等效分 | 考生位次（与今年相同） | 用考生位次反查去年 rank 表 |
| 前年（如 2024） | 前年等效分 | 考生位次（与今年相同） | 用考生位次反查前年 rank 表 |

### 趋势解读（1-2 段）

- **分数走势**：与今年分差反映该位次在不同年份的"难度"
- **竞争激烈程度**：去年等效分 < 今年分，说明今年题难 / 竞争激烈；反之亦然
- **近年波动**：等效分差距越小，说明该位次越稳定

**降级策略**：分数缺失则本节直接跳过；去年或前年 rank 表缺失则只展示有的年份。

## 2. 志愿筛选（按等效分锚定）

> **辽宁志愿模式速记**：每个 (院校, 专业) 组合 = 1 个独立志愿槽。本方案不再使用"冲 / 稳 / 保"档位概念，所有候选志愿按"**代表位次升序**"排列（位次小=门槛高=排前；位次大=门槛低=排后）。
>
> **核心方法**：以等效分 `E` 为锚点，SQL 提取候选池 → 合并去重 ~1000 个 → AI 凭常识评分 → **以考生位次为分界，向上向下各取综合分 top 100 → 合并 200 条后按代表位次升序排列**（即志愿填报顺序）。
>
> **为什么按代表位次升序**：辽宁"分数优先、遵循志愿"按志愿表顺序检索——**第一个分数够的就录取**，所以代表位次小（门槛高）的志愿必须排前面（虽然今年分数够不上会跳过），不能把代表位次大的志愿排到前面。

### 划线原理

**不能用"（去年等效分 + 前年等效分）/ 2"作为基准分**——平均后的分数在历史上从未存在过，对应的"录取格局"也无从查证。

**正确做法**：

1. 用考生今年位次反查去年 rank 表 → 得到**去年等效分**（记为 `E_去年`）
2. 用考生今年位次反查前年 rank 表 → 得到**前年等效分**（记为 `E_前年`）
3. 以 `E_去年` 和 `E_前年` 为锚点，从去年、前年录取分表中各提取上下 N 个志愿（默认 N=300，可调整到 500）
4. 4 张候选表合并去重（按 (院校, 专业)）→ 估计剩 800-1200 个志愿
5. AI 基于考生偏好凭常识评分 → 计算每个候选志愿的**综合分**（0-10）
6. **志愿推荐表选取**：以考生今年位次 `P` 为分界
   - **向上 100 条**：代表位次 < `P` 的候选中，**按综合分降序**取前 100
   - **向下 100 条**：代表位次 ≥ `P` 的候选中，**按综合分降序**取前 100
   - 合并 200 条 → 按"**代表位次升序**"排列（位次小=门槛高=排前）
   - **注意**：这里的 "top 100" 是按 AI 综合分（推荐分数），**不是按实际录取分数**

### 2.1 合并去重（4 张候选表 → ~1000 个志愿）

> SQL 候选池是中间过程，不在本报告展示。SQL 取数步骤详见 [AGENTS.md 6.9](../AGENTS.md)。

以等效分 `E_去年` / `E_前年` 为锚点，从去年、前年录取分表中各取上下 N 行（默认 N=300），共 **4 张候选表**；按 (院校, 专业) 合并去重后，估计剩 **800-1200 个志愿**。

**去重规则**：
- 同一 (院校, 专业) 在多张表出现 → 合并到一行，保留完整字段（去年分 + 前年分 + 今年计划数 + 双方位次）
- 只有一年有数据的志愿 → 另一年标 "—"
- 同一专业不同代码 / 不同方向（如"辽宁大学 临床医学"多个方向）→ 保留多行

**降级策略**：某张候选表返回行数 < N：如实保留，不补。

### 2.2 AI 评分原则（本次考生临时设计）

> **重要**：评分维度、权重、范围根据本次考生情况（分数、选科、偏好）临时设计，**不写死**到模板里。本节由 AI 在生成方案时根据 question.md 填写内容自行确定。

**通用指引**（AI 设计评分原则时应考虑的方向，**不是硬性清单**）：

| 维度方向 | 说明 |
|---|---|
| 学校层次 | C9 / 985 / 211 / 双一流 / 一本 / 普通本科 |
| 专业热度 | 基于考生想学专业加权（如计算机、电子信息） |
| 地理位置 | 基于考生位置偏好加权（如辽宁周边、京津沪） |
| 计划数稳定性 | plan_num 过小（< 2）减权 |
| 中外合作 | 学费高（一年 4-6 万），通常扣分 |
| 特殊行业 | 农林地矿等，部分考生/家长不接受，通常扣分 |
| 其他 | AI 认为对当前考生有意义的维度 |

**关键原则**：
- **AI 凭常识逐项打分**，不硬编码（如不写 `CITY_SCORES = {"北京": 10, ...}` 这种大字典）
- **不列出每个维度分数**（评分原则已说明，避免大表冗长）
- 综合分用于"志愿推荐表选取"（按位次分界上下各取 top 100），**不代表录取概率**——综合分高 = AI 推荐度强

**排序键**：
- 主键：**代表位次升序**（位次小=门槛高=排前，符合辽宁检索逻辑）
- 同分 tie-breaker 1：计划数大优先
- 同分 tie-breaker 2：综合分高优先（AI 推荐度）

**代表位次定义**：min(去年位次, 前年位次)（两年都有数据时）。

**分批打分**（推荐做法，可调整）：

> 本次评分采用**分批打分**机制，共分 **N 批**，**每批 X 条志愿**。
> 每批开始前，AI 用 2 个"标定志愿"（如清华计算机、某偏远二本）快速重打，
> **校准跨批次分数尺度**，避免一次性处理 ~1000 个志愿时的 context 压力、
> 注意力衰减与输出控制问题。

**AI 评分承诺**：

> 本节综合分与评语由 AI **逐一查看**每一个 (院校, 专业) 组合后凭常识判断给出，
> **未使用**任何硬编码字典、规则脚本或通用模板。
> 每条评语含可核实事实（学科评估、招生人数、招生章程条款等），不是空泛套话。
> 详见 [附录 A.3](#a3-ai-评分承诺防投机取巧)。

#### 志愿推荐表选取规则（默认 200 条）

> **核心原则**：不以"综合分降序取 Top N"作为报告主表筛选条件——
> 综合分高的志愿往往代表位次小（门槛高），全集中在冲段，违反辽宁"分数优先、
> 遵循志愿"检索逻辑下"志愿表必须有命中区志愿"的基本要求。

**选取逻辑**：

```
考生今年位次 P（分界点）
   │
   ├── 向上 N 条：代表位次 < P 的候选中，按 AI 综合分降序取前 N
   │
   └── 向下 N 条：代表位次 ≥ P 的候选中，按 AI 综合分降序取前 N

合并 2N 条 → 按代表位次升序排列（即志愿填报顺序）
```

**关键提醒**：

- **"top N" 是按 AI 综合分（推荐分数）排序**，**不是按实际录取分数**
- 综合分反映"该志愿对考生的推荐价值"，与"该志愿的实际录取难度（代表位次）"是不同维度
- N 默认 = 100，推荐表 = **200 条**（覆盖完整梯度）
- 命名：不叫 "Top N"——"Top" 隐含"最优/最前列"，但 N 条是覆盖完整梯度的"推荐"，不是"客观最优"
- 报告主表正式名称：**志愿推荐表**（不是 "Top 200" / "Top 150" 等）

**N 调参建议**：

| 考生场景 | N 取值 | 推荐表数量 | 理由 |
|---|---|---|---|
| 600+ 高分段 | 80-100 | 160-200 | 高分志愿稀疏 |
| 500-600 主流分段 | 100 | 200 | **默认** |
| 450- 中低分段 | 100-120 | 200-240 | 低分段志愿密集 |
| 考生偏好宽（不限地区/专业） | 120-150 | 240-300 | 候选池大 |
| 考生偏好窄（指定专业+城市） | 60-80 | 120-160 | 候选池本身已小 |

**边界处理**：
- 某方向不足 N 条：取全部，不补（如实展示，不硬填）
- 某方向超过 N 条：按综合分降序取前 N
- 候选总池不足 2N 条：如实展示全部，不硬填

**完整方法论沉淀**：[AGENTS.md 6.8.2 决策记录](../AGENTS.md#682-决策记录从top-150-综合分降序到志愿推荐表向上-100--向下-100)。

### 2.3 志愿推荐表（默认 200 条）

> **筛选流程**：以考生今年位次 `P` 为分界
> - **向上 N 条**：代表位次 < `P` 的候选中，按 AI 综合分降序取前 N
> - **向下 N 条**：代表位次 ≥ `P` 的候选中，按 AI 综合分降序取前 N
> - **合并 2N 条 → 按代表位次升序排列**（即志愿填报顺序）
>
> **重要**：这里的 "top N" 是按 AI 综合分（推荐分数），**不是按实际录取分数**。
> AI 综合分反映"该志愿对考生的推荐价值"，与"实际录取难度（代表位次）"是不同维度。
>
> **默认 N=100，推荐表 200 条**——覆盖从高分冲段（< P）到低分保段（> P）的完整梯度。
> 家长可按本表顺序挑出 112 个填入辽宁志愿系统（辽宁志愿表容量 112 槽）。
>
> **为什么按代表位次升序**：辽宁"分数优先、遵循志愿"按志愿表顺序检索——**第一个分数够的就录取**，所以代表位次小的志愿必须排前面（虽然今年分数够不上会跳过），不能把代表位次大的志愿排到前面。

| 序号 | 代表位次 | 院校 | 专业 | 计划数（今年） | 前年（如 2024）分/位次 | 去年（如 2025）分/位次 | 综合分 | 评语 |
|---|---|---|---|---|---|---|---|---|
| 1 | | | | | | | | |
| 2 | | | | | | | | |
| ... | | | | | | | | |
| 200 | | | | | | | | |

**字段说明**：
- **代表位次**：min(去年位次, 前年位次)，位次小=门槛高=排前
- **前年分/位次 + 去年分/位次**：两列原始数据已直接体现波动，**不再单独列"位次波动"列**
- **综合分**：AI 评分结果（0-10），仅在本推荐表（默认 200 条）内有效，不代表录取概率
- **评语**：AI 凭常识判断（含专业热度、学校特色、稳定性等定性描述）

- 最有命中可能的志愿（代表位次 ≈ 考生位次）排在中间位置（不是第 1 个）
- 评语栏写专业方向、特殊要求、是否新设/停招等提示信息

> **格式约定**：
> - 编码：UTF-8
> - 字段分隔符：英文逗号 `,`
> - 包含逗号 / 引号 / 换行的字段用 `"..."` 包裹，引号转义用 `""`
> - 缺失值（无该年数据）：`—`（中文破折号 U+2014）
> - 排序：与志愿推荐表一致，按代表位次升序
> - 与 Markdown 志愿推荐表的差异：CSV 把"前年分/位次"拆成两列，方便 Excel 排序 / 筛选

### A.3 AI 评分承诺（防投机取巧）

> **本报告 2.3 节志愿推荐表（默认 200 条）与附录 A 中两个 CSV 文件的"综合分"与"评语"列，由 AI 大模型逐一查看每一个 (院校, 专业) 组合后凭常识判断给出，**未使用任何硬编码字典、规则脚本或通用模板**。**

**分批打分**：

> 本次评分采用**分批打分**，共 **N 批**，**每批 X 条志愿**。
> 每批开始前 AI 用 2 个"标定志愿"快速重打，**校准跨批次分数尺度**，
> 避免一次性处理 ~1000 个志愿时的 context 压力、注意力衰减与输出控制问题。

**评语质检**：
- 每条评语都针对该志愿的具体情况，**含可核实事实**（学科评估、招生人数、招生章程条款等）
- 评语不是空泛的"好学校""符合偏好"等套话
- 评语不是从其他志愿复制粘贴

如发现评语空泛、与该志愿无关、或为通用模板套用，请反馈至项目 GitHub Issues。

### A.4 使用方式

- **家长**：用 Excel / WPS 打开 CSV
  - 按"综合分"列降序排序 → 看 AI 推荐
  - 按"代表位次"列升序排序 → 看填报顺序
  - 按"院校"列筛选 → 单一院校横向对比
- **AI 复核**：可让 AI 重新读取 CSV 做交叉验证或进一步分析
