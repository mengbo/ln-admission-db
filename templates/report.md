# 志愿填报报告

> 本模板用于 AI Agent 在收到考生信息后一次性生成完整志愿报告。
> 拿到 [question.md](./question.md) 填写完的内容后，按本模板的章节顺序输出。
>
> 报告与具体年份无关：
> - "今年" = 考生参加高考的年份（AI 从考生信息中获取）
> - "去年" = 今年 − 1
> - "前年" = 今年 − 2
> - 历史分数与计划数取自 `score_phyYYYY` / `score_hisYYYY` / `plan_YYYY`，YYYY 对应实际年份
>
> **下方括号内的年份（如 2026 / 2025 / 2024）均为示例，AI 须按考生实际高考年份替换为对应今年/去年/前年。**

---

## 报告头

- 考生姓名 / 高考年份 / 首选科目 / 选科组合
- 高考分数 + 对应**今年（如 2026）一分一段位次**（核心数据，贯穿全报告）
- 报告生成日期

## 1. 定位分析

> 等效分换算的原理与计算步骤见 [AGENTS.md 数据库分析经验](../AGENTS.md#数据库分析经验)。

### 等效分换算表

| 年份 | 分数 | 位次 | 说明 |
|---|---|---|---|
| 今年（如 2026） | 考生原始分 | 考生位次（从今年 rank 表查） | 直接给出 |
| 去年（如 2025） | 去年等效分 | 考生位次（与今年相同） | 用考生位次反查去年 rank 表 |
| 前年（如 2024） | 前年等效分 | 考生位次（与今年相同） | 用考生位次反查前年 rank 表 |

### 趋势解读（1-2 段）

- **分数走势**：与今年分差反映该位次在不同年份的"难度"
- **竞争激烈程度**：去年等效分 < 今年分，说明今年题难 / 竞争激烈；反之亦然
- **近年波动**：等效分差距越小，说明该位次越稳定

**降级策略**：分数缺失则本节直接跳过；去年或前年 rank 表缺失则只展示有的年份。

## 2. 冲稳保志愿清单

> 辽宁平行志愿可填 100+ 个志愿。
> **核心方法**：分别用去年、前年的实际录取数据各自筛选志愿，再综合两年结果得出最终冲稳保。
> 整个决策过程在报告中透明呈现，便于家长复核。

**志愿数量建议**：

| 档次 | 建议志愿数 |
|---|---|
| 冲 | 20 |
| 稳 | 20 |
| 保 | 20 |
| 合计 | 60 |

### 划线原理

**不能用"（去年等效分 + 前年等效分）/ 2"作为基准分**——平均后的分数在历史上从未存在过，对应的"录取格局"也无从查证。

**正确做法**：分别用两年数据独立划线，再综合：

1. 用考生今年位次反查去年 rank 表 → 得到**去年等效分**（这是"去年录取格局下考生相当于多少分"）
2. 用考生今年位次反查前年 rank 表 → 得到**前年等效分**（这是"前年录取格局下考生相当于多少分"）
3. **去年角度**：把去年等效分 ±15 作为分界线，看每个志愿的去年实际录取分落在哪档
4. **前年角度**：把前年等效分 ±15 作为分界线，看每个志愿的前年实际录取分落在哪档
5. **综合**：取两年中**较保守**的档位（避免高估录取难度，确保志愿表的安全垫）

### 2.C 两年综合

**综合规则（取保守档位）**：冲 > 稳 > 保，**取两者中更偏"保"的那一档**。

| 去年档位 | 前年档位 | 综合档位 | 解读 |

| 冲 | 稳 | 稳 | 取保守 |
| 稳 | 冲 | 稳 | 取保守 |
| 冲 | 保 | 保 | 取保守 |

| 稳 | 稳 | 稳 | |

- 两年都缺失：综合判定为"未知"，AI 需单独标注并人工判断
- 偏好为空时按物理类/历史类通用热门专业排序
- 候选不足时每档可暂时少于建议数，并标注"建议补充 X 信息后扩充"

|---|---|---|---|---|---|---|
| | | | | | | 低/中/高 |

**主要风险点**（5-8 条文字）：
- 例：XX 大学 XX 专业前年（如 2024）停招，去年（如 2025）新设，今年（如 2026）需重点关注
- 例：XX 大学计划数从 30 缩减到 15，分数线可能上浮
