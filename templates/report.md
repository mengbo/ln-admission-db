# 志愿填报方案

> 本模板用于 AI Agent 在收到考生信息后一次性生成完整志愿填报方案。
> 拿到 [question.md](./question.md) 填写完的内容后，按本模板的章节顺序输出。
>
> 方案与具体年份无关：
> - "今年" = 考生参加高考的年份（AI 从考生信息中获取）
> - "去年" = 今年 − 1
> - "前年" = 今年 − 2
> - 历史分数与计划数取自 `score_phyYYYY` / `score_hisYYYY` / `plan_YYYY`，YYYY 对应实际年份
>
> **下方括号内的年份（如 2026 / 2025 / 2024）均为示例，AI 须按考生实际高考年份替换为对应今年/去年/前年。**
>
> **关于名称**：本项目把"志愿填报报告 / 志愿填报指南 / 志愿方案 / 报告 / 方案 / 指南"都视为同义词，最终输出物的正式名称是 **"志愿填报方案"**。详见 [AGENTS.md 术语约定](../AGENTS.md#-术语约定)。

## 1. 定位分析

> 等效分换算的原理与计算步骤见 [AGENTS.md 数据库分析经验](../AGENTS.md#数据库分析经验)。

### 等效分换算表

| 年份 | 分数 | 位次 | 说明 |
|---|---|---|---|
| 今年（如 2026） | 考生原始分 | 考生位次（从今年 rank 表查） | 直接给出 |
| 去年（如 2025） | 去年等效分 | 考生位次（与今年相同） | 用考生位次反查去年 rank 表 |
| 前年（如 2024） | 前年等效分 | 考生位次（与今年相同） | 用考生位次反查前年 rank 表 |

### 趋势解读（1-2 段）

- **分数走势**：与今年分差反映该位次在不同年份的"难度"
- **竞争激烈程度**：去年等效分 < 今年分，说明今年题难 / 竞争激烈；反之亦然
- **近年波动**：等效分差距越小，说明该位次越稳定

**降级策略**：分数缺失则本节直接跳过；去年或前年 rank 表缺失则只展示有的年份。

## 2. 志愿筛选（按等效分锚定）

> **辽宁志愿模式速记**：每个 (院校, 专业) 组合 = 1 个独立志愿槽，辽宁共可填 **112 个**志愿。本方案不再使用"冲 / 稳 / 保"档位概念，所有候选志愿按"代表录取分降序"排列（高分在前、低分在后）。
>
> **核心方法**：以等效分 `E` 为锚点，SQL 提取候选池 → 合并去重 → 大模型软筛选 → 保留 112 个代表性志愿。
>
> **为什么按降序**：辽宁"分数优先、遵循志愿"按志愿表顺序检索——**第一个分数够的就录取**，所以高分志愿必须排前面（虽然够不上会跳过），不能把低分志愿排到前面。

### 划线原理

**不能用"（去年等效分 + 前年等效分）/ 2"作为基准分**——平均后的分数在历史上从未存在过，对应的"录取格局"也无从查证。

**正确做法**：

1. 用考生今年位次反查去年 rank 表 → 得到**去年等效分**（记为 `E_去年`）
2. 用考生今年位次反查前年 rank 表 → 得到**前年等效分**（记为 `E_前年`）
3. 以 `E_去年` 和 `E_前年` 为锚点，SQL 提取上下各 N 个志愿（默认 N=300，可调整到 500）
4. 合并去重（按 (院校, 专业)）→ 大模型基于 question.md 软筛选 → 保留 112 个
5. 按"代表录取分降序"排列（高分在前、低分在后）

|---|---|---|---|---|---|

### 2.2 合并去重后 ~1000 个志愿

（汇总：4 张表合并后按 (院校, 专业) 去重 → 估计剩 800-1200 个志愿）

**去重规则**：
- 同一 (院校, 专业) 在多张表出现 → 合并到一行，保留完整字段（去年分 + 前年分 + 今年计划数 + 双方位次）
- 只有一年有数据的志愿 → 另一年标 "—"
- 同一专业不同代码 / 不同方向（如"辽宁大学 临床医学"多个方向）→ 保留多行

### 2.3 大模型软筛选 → 保留 112 个

#### 软筛选原则（基于 question.md，不硬编码到大模型 prompt）

| 维度 | 处理 |
|---|---|
| 学校位置 | 考生写"600− 倾向辽宁周边" → 减权偏远地区；考生写"600+ 全国不限" → 不做位置筛选 |
| 想学专业 | 考生写"计算机"等 → 加权相关专业志愿 |
| 不想学的专业 | 考生写"医学生"等 → 减权或淘汰 |
| 家庭就业资源 | 考生写"父亲 IT 行业" → 加权 IT 类志愿 |
| 计划数过小 | plan_num < 2 → 减权或淘汰 |
| 录取分波动大 | 两年分差 > 30 → 减权或淘汰 |
| 同分志愿排序 | tie-breaker：波动小优先 → 计划数大优先 → 符合偏好优先（详见 AGENTS.md 6.9.3） |

**保留数量**：112 个（= 辽宁平行志愿总槽位数）。如果软筛选后保留 < 112：如实返回，不补。

| 序号 | 院校 | 专业 | 计划数（今年） | 前年（如 2024）分/位次 | 去年（如 2025）分/位次 | 两年波动 | 备注 |

| 1 | | | | | | | |
| 2 | | | | | | | |
| ... | | | | | | | |
| 112 | | | | | | | |

**填表说明**：
- 按上表顺序逐个填入辽宁志愿系统（前 1~112）
- 录取分降序是检索逻辑决定的，**不是偏好**——高分志愿排前面虽然够不上会跳过，但必须排在前面，因为考生分数够高时会从这些志愿开始命中
- 最有命中可能的志愿（录取分 ≈ 等效分 E）排在中间位置（不是第 1 个）
